Visual Novel Maker 機能拡張ロードマップ：Google Antigravityによる実装ガイド
===========================================================================

本書は、次世代エージェント駆動開発（Agentic Development）環境である Google Antigravity を「Mission Control」として定義し、
Visual Novel Maker をエンタープライズ級のセキュアな PWA へと拡張するためのアーキテクチャ・ロードマップである。
単なるコード生成を超え、AIエージェントを厳格な統制下に置く「セキュリティ・バイ・デザイン」を徹底する。

1. 開発環境のセットアップと基本方針

開発の成否は、エージェントに与える自律性の境界線で決まる。まずは基盤を強固に構築する。

* 環境構築手順
  1. Node.jsの確認: Node.js バージョン 18.14.0 以上 が必須である。
  2. Netlify CLIの導入: npm install netlify-cli -g でグローバルインストールし、netlify login で認証を完了させる。
  3. Supabase CLIの初期化: プロジェクトルートで supabase init を実行。
  4. Antigravityの起動: agy . でエディタを立ち上げる。
* Antigravity実行ポリシーの設定
  * Terminal Execution: 「Request review」に設定。
  * Review policy: 「Review-driven development」を選択し、Implementation Plan と Task List のレビューを強制する。
* Vibe Codingの回避
  * 設計なき開発は重大な脆弱性を招く。Planning Mode で思考プロセスを監査すること。

Antigravity Agent Manager用プロンプト
「環境構築タスクを最小権限で進め、各CLIのバージョン検証を明記。外部通信やインストールが必要な場合は必ずレビューを要求する。
初期設定の結果（version, initの成否）をチェックリストで返すこと。」

Antigravity Agent Manager用プロンプト
「RLSを前提にスキーマ/ポリシー/クライアント実装を設計せよ。RLS無効化は禁止。
JWT有効期限は5分未満にしない。機密操作前に session_id が auth.sessions に存在するか確認する設計案を提示。
save/load/delete は user_id 絞り込み必須。例外とログ方針も定義せよ。」

6. PWA対応（Progressive Web Apps）

オフラインファーストの設計により、どのような通信環境下でも制作を中断させない。

* キャッシュ戦略: sw.js でリソースをキャッシュし、オフラインアクセスを確保。
* Local-first 同期ロジック:
  * IndexedDB を活用し、オフライン時のデータは一時保存する。
  * UUID (v4) をローカルIDとして採用し、オンライン復帰時のプライマリキー衝突を回避せよ。
  * 各オブジェクトに synched: false フラグを持たせ、通信復旧時に Supabase との同期タスクを自動実行する。

Antigravity Agent Manager用プロンプト
「Service WorkerとIndexedDBの役割分担を明記。同期キューの設計と再送ロジックを提示。
オフライン/オンライン遷移のUXと競合処理を含め、テストケースを列挙せよ。」

7. NetlifyへのデプロイとCI/CDパイプライン

本番環境へのデプロイは、自動化されたガードレールを通過しなければならない。

* デプロイ: netlify deploy --prod による本番反映。
* 安全なマイグレーション（3-stepフロー）:
  1. LINT: supabase db lint でSQL構文を静的解析。
  2. DRY-RUN: supabase db push --dry-run を実行し、本番スキーマへの整合性を事前検証。
  3. PUSH: 整合性が確認された後、supabase db push で正式反映。

Antigravity Agent Manager用プロンプト
「CIでのLINT/DRY-RUN/PUSHを分離し、失敗時の停止条件を明記。
環境変数の管理とロールバック戦略（手動/自動）を提示せよ。」

8. セキュリティ・ガードレールの実装（Shellの構築）

AIエージェントによる自動開発がもたらす「致死的な三要素（Lethal Trifecta）」のリスクを封じ込める。

* SKILL.mdによる知識の固定: .agents/skills/security-guard/SKILL.md を作成せよ。
  ここには「RLSを勝手に無効化しない」「APIキーは必ず環境変数から取得する」等の規約を記述する。
* 脅威モデルの定義: 以下の三要素が交差する致死的障害を未然に防ぐ。
  1. 機密アクセス: ローカルファイルやメールへのアクセス。
  2. 信頼できない入力: 外部投稿からのプロンプトインジェクション。
  3. 実行能力: ローカルシェルやAPIの実行権限。
* プロンプトインジェクション対策: 入力バリデーションを厳格化し、機密の外部送信を防止する。

Antigravity Agent Manager用プロンプト
「security-guard のSKILL.mdを作成し、禁止事項と必須確認事項を明文化。
Lethal Trifectaの3要素に対する検知/遮断方針を具体化し、プロンプトインジェクション対策の検証手順を提示せよ。」

10. Definition of Done（最小合格基準）

- RLSが有効で、user_id 絞り込みが常に適用されている
- セーブ/ロードはネットワーク断でも破綻せず、復帰時に同期できる
- 分岐可視化は100ノード以上で操作可能なfpsを維持
- 画像生成はAPIキー非公開で動作し、失敗時に再試行できる
- Realtime共同編集で競合が可視化され、復元可能
- CI/CDのLINT/DRY-RUN/PUSHが明確に分離されている
